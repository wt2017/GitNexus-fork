Architecture: GitNexus

Overview

GitNexus is a graph-powered code intelligence platform that indexes any codebase into a queryable knowledge graph and exposes it to AI agents via MCP (Model Context Protocol) and a web-based visual explorer. It parses source code using tree-sitter, builds a property graph of symbols and relationships, detects communities and execution processes, and optionally generates semantic embeddings for natural-language search.

The project is organized as a monorepo with a Node.js CLI / server package, a React web UI, shared TypeScript definitions, and IDE plugin integrations for Claude and Cursor.

System Architecture

GitNexus follows a CLI-first, server-optional architecture. The core engine runs as a local Node.js CLI that can also start an HTTP server. The server hosts both a REST API for the web UI and an MCP-over-HTTP endpoint for remote AI tools. A separate web container serves the static React SPA.

Data Flow

  1. Ingestion: The user runs gitnexus analyze <path> (or via the web UI). The ingestion pipeline walks the filesystem, parses each source file with tree-sitter, extracts symbols (functions, classes, variables, imports), resolves references, and writes nodes and relationships into LadybugDB.
  2. Community & Process Detection: After symbol extraction, the pipeline runs community detection (Leiden) to group related symbols into functional areas, and process detection to identify execution flows (e.g., request handlers, middleware chains).
  3. Indexing: Metadata (meta.json) and the registry (~/.gitnexus/registry.json) are updated so the MCP server and HTTP API can discover the repo.
  4. Querying: AI agents issue MCP tool calls (query, context, impact, detect_changes) or the web UI calls REST endpoints (/api/graph, /api/search, /api/file). The LocalBackend translates these into Cypher queries against LadybugDB.
  5. Embeddings (optional): When --embeddings is enabled, the embedding pipeline generates vector representations of symbol content using a local ONNX model and stores them in LadybugDB for semantic search.

Key Modules

CLI Layer

Purpose: Parses user commands and dispatches to lazy-loaded action handlers. Supports analyze, serve, mcp, query, context, impact, wiki, and maintenance commands.

Entry point: gitnexus/src/cli/index.ts

Key files: cli/index.ts, cli/analyze.ts, cli/serve.ts, cli/mcp.ts, cli/tool.ts, cli/lazy-action.ts

Ingestion Pipeline

Purpose: Core static analysis engine. Parses source files into ASTs, extracts symbols and relationships, resolves imports/calls, detects communities and processes, and writes the graph to LadybugDB. Supports incremental indexing via content-hash diffing.

Entry point: gitnexus/src/core/ingestion/pipeline.ts

Key files: core/ingestion/pipeline.ts, core/ingestion/parsing-processor.ts, core/ingestion/community-processor.ts, core/ingestion/process-processor.ts, core/ingestion/import-processor.ts, core/ingestion/call-processor.ts, core/ingestion/workers/

LadybugDB Adapter

Purpose: Abstracts all graph database operations: connection pooling, Cypher execution, schema initialization, WAL flushing, and incremental node deletion. Replaced KuzuDB in earlier versions.

Entry point: gitnexus/src/core/lbug/lbug-adapter.ts

Key files: core/lbug/lbug-adapter.ts, core/lbug/pool-adapter.ts, core/lbug/schema.ts, core/lbug/lbug-config.ts

Storage & Repo Manager

Purpose: Manages on-disk storage for indexes (.gitnexus/), the global registry (~/.gitnexus/registry.json), parse caches, file hashes for incremental updates, and git integration.

Entry point: gitnexus/src/storage/repo-manager.ts

Key files: storage/repo-manager.ts, storage/git.ts, storage/parse-cache.ts, storage/file-hash.ts

Search & Embeddings

Purpose: Provides keyword search (BM25/FTS), semantic search (cosine similarity on ONNX embeddings), and hybrid ranking. The embedding pipeline runs locally without cloud dependencies.

Entry point: gitnexus/src/core/search/hybrid-search.ts

Key files: core/search/hybrid-search.ts, core/search/bm25-index.ts, core/embeddings/embedder.ts, core/embeddings/embedding-pipeline.ts, core/embeddings/exact-search.ts

MCP Server

Purpose: Implements the Model Context Protocol for AI agent integration. Exposes tools (query, context, impact, detect_changes, cypher, rename), resources (repo context, clusters, processes, schema), and prompts (detect_impact, generate_map). Runs over stdio or StreamableHTTP.

Entry point: gitnexus/src/mcp/server.ts

Key files: mcp/server.ts, mcp/tools.ts, mcp/resources.ts, mcp/local/local-backend.ts, mcp/compatible-stdio-transport.ts

HTTP API Server

Purpose: Express-based REST API serving the web UI and external clients. Endpoints include graph streaming (NDJSON), search, file reading, grep, process/cluster queries, analysis job management (with SSE progress), and embedding jobs. Serves the built web UI as a static SPA.

Entry point: gitnexus/src/server/api.ts

Key files: server/api.ts, server/analyze-job.ts, server/analyze-worker.ts, server/git-clone.ts, server/mcp-http.ts, server/validation.ts

Web UI

Purpose: React 19 SPA for visual exploration of the knowledge graph. Features: interactive Sigma.js graph canvas, file tree panel, code/chat side panel, LLM agent chat, settings, and repo onboarding (drag-drop or server connect). Connects to the HTTP API.

Entry point: gitnexus-web/src/main.tsx

Key files: gitnexus-web/src/App.tsx, gitnexus-web/src/components/GraphCanvas.tsx, gitnexus-web/src/services/backend-client.ts, gitnexus-web/src/hooks/useAppState.tsx, gitnexus-web/src/core/graph/graph.ts, gitnexus-web/src/core/llm/agent.ts

Shared Types

Purpose: Shared TypeScript type definitions used by both the CLI and web UI packages to ensure contract consistency across the monorepo.

Entry point: gitnexus-shared/src/index.ts

Key files: gitnexus-shared/src/index.ts, gitnexus-shared/src/graph/, gitnexus-shared/src/integrations/, gitnexus-shared/src/test-helpers.ts

External Dependencies

  • Git — Required for repo cloning, commit detection, staleness checks, and diff parsing.
  • LadybugDB — Embedded graph database (via @ladybugdb/core npm package). No external DB server needed.
  • Hugging Face Transformers — Local embedding model download and inference (ONNX-based).
  • LLM APIs (optional) — OpenAI, OpenRouter, Azure, Cursor for wiki generation and agent chat in the web UI.
  • GitHub (optional) — For publishing wiki gists and the understand-quickly registry integration.

Configuration

Key configuration files and environment variables:

  • ~/.gitnexus/config.json — Global CLI config (API keys, model preferences). Written with 0o600 permissions.
  • ~/.gitnexus/registry.json — Global repo registry mapping names to indexed repo paths.
  • .env.example — Docker Compose environment variables:
    • SERVER_IMAGE / WEB_IMAGE — Container image references
    • SERVER_HOST_PORT (default 4747) / WEB_HOST_PORT (default 4173)
    • WORKSPACE_DIR — Optional read-only mount for repos to index
  • CLI environment variables: GITNEXUS_HOME, GITNEXUS_NO_GITIGNORE, GITNEXUS_MAX_FILE_SIZE, GITNEXUS_EMBEDDING_THREADS, etc.
